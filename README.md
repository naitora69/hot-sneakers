# hot-sneakers
### Проект представляет собой микросервисную платформу для проведения онлайн-аукционов редких кроссовок в режиме реального времени.
### Архитектура построена на принципах гексагональной архитектуры, обеспечивая изоляцию бизнес-логики от инфраструктурных зависимостей и высокую тестируемость системы.
# Технологический стек

## Язык и базовые технологии

- **Go (Golang)** — основной язык разработки микросервисов.
- **net/http** — маршрутизация и обработка HTTP-запросов.
- **gorilla/websocket** — двусторонняя realtime-коммуникация с клиентами.

---

## API и Межсервисное взаимодействие

- **gRPC + Protocol Buffers (`protoc`)** — бинарное взаимодействие между внутренними сервисами.
- **RESTful API** — публичный интерфейс через API Gateway.
- **NATS (Core)** — событийная шина для асинхронного обмена сообщениями.

---

## Базы данных и Хранение

- **PostgreSQL** — основное реляционное хранилище.
- **Redis** — in-memory кеш для ускорения чтения каталога.
- **pgx + sqlx** — драйвер и инструменты для эффективной работы с SQL.

---

## Конфигурация

- **cleanenv** — строгая типизация и загрузка конфигураций.

---

## Тестирование

- **Integration Tests**
---

## Инфраструктура и CI/CD

- **GitHub Actions**
- **Docker & Docker Compose**

---

### Приложение состоит из 5 основных компонентов

## 1) api

REST API Gateway, принимающий HTTP-запросы от клиентов и маршрутизирующий их во внутренние сервисы через gRPC.

### Что было сделано

Реализовал централизованный API Gateway с маршрутизацией запросов и агрегацией ответов внутренних сервисов.

Внедрил:

- **JWT middleware**
- **Rate Limiting**
- **Graceful Shutdown**
- **Health Check**
- маппинг gRPC ошибок в HTTP-статусы

Для защиты системы под высокой нагрузкой применил паттерны:

- **Proxy**
- **Semaphore**
- **Fan-out / Fan-in**

---

## 2) catalog

CRUD-сервис управления объявлениями о продаже кроссовок.

### Что было сделано

Спроектировал сервис каталога с поддержкой создания, редактирования и удаления объявлений.

Для оптимизации производительности внедрил **Cache-Aside стратегию** с использованием Redis.

Это позволило:

- существенно снизить количество обращений к PostgreSQL;
- ускорить получение популярных карточек товаров;
- уменьшить latency чтения каталога.

---

## 3) auction

Ключевой сервис платформы, отвечающий за проведение торгов и обработку ставок.

### Что было сделано

Реализовал конкурентно-безопасную обработку ставок через транзакционный механизм PostgreSQL.

Для обеспечения консистентности использовал:

- уровень изоляции **Read Committed**
- пессимистические блокировки через **SELECT FOR UPDATE**

Это позволило устранить проблему **Lost Update** и гарантировать корректную последовательность обработки конкурентных ставок.

Дополнительно реализовал:

- валидацию ставок;
- контроль жизненного цикла аукциона;

---

## 4) notification

Сервис realtime-уведомлений.

### Что было сделано

Спроектировал событийный механизм мгновенной доставки уведомлений пользователям через WebSocket.

Интегрировал сервис с **NATS**, обеспечив асинхронную публикацию событий о:

- новых ставках;
- завершении торгов;
- изменении статуса аукциона.

Это позволило обеспечить актуальное состояние интерфейса без polling.

---

## 5) login

Сервис аутентификации и авторизации.

### Что было сделано

На данный момент находиться в стадии разработки.

---

# Ключевые инженерные решения

## Кеширование каталога

Внедрил **Redis Cache-Aside**, что позволило разгрузить PostgreSQL и ускорить чтение популярных товаров.

---

## Конкурентная обработка ставок

Критический сценарий торгов реализован через транзакции и row-level locking.

Это гарантирует:

- отсутствие race conditions;
- честную обработку ставок;
- строгую последовательность обновления цены.

---

## Event-Driven архитектура

Перевел взаимодействие между auction и notification на событийную модель через **NATS**.

Это обеспечило:

- минимальную задержку доставки событий;
- слабую связанность сервисов;
- масштабируемость realtime-подсистемы.

---

## CI/CD и тестирование

Самостоятельно настроил **GitHub Actions pipeline**, включающий:

- линтеры;
- unit tests;
- integration tests;
- автоматическую сборку контейнеров.

Вся инфраструктура контейнеризирована и запускается через **Docker Compose**, обеспечивая идентичность dev/test окружений.

# API Quick Start

## Health Check

Проверка доступности gateway и зависимых сервисов:

```bash
curl http://localhost:38080/api/ping
```

---

## Auction API

### Получить информацию об аукционе

```bash
curl http://localhost:38080/api/auction?id=1
```

---

### Создать новый аукцион

```bash
curl -X POST http://localhost:38080/api/auction/create \
  -H "Content-Type: application/json" \
  -d '{
    "sneaker_id": 1,
    "start_price": 500,
    "duration_minutes": 60
  }'
```

---

### Сделать ставку

```bash
curl -X POST http://localhost:38080/api/auction/bid \
  -H "Content-Type: application/json" \
  -d '{
    "auction_id": 1,
    "user_id": 42,
    "amount": 750
  }'
```

---

## Catalog API

### Получить список всех кроссовок

```bash
curl http://localhost:38080/api/sneakers
```

---

### Получить карточку кроссовок по ID

```bash
curl http://localhost:38080/api/sneaker?id=1
```

---

### Создать новое объявление

```bash
curl -X POST http://localhost:38080/api/sneaker/create \
  -H "Content-Type: application/json" \
  -d '{
    "brand": "Nike",
    "model": "Air Jordan 1 Retro High",
    "size": 43,
    "condition": "deadstock",
    "price": 1200
  }'
```
## Подключиться к WebSocket уведомлениям

```bash
ws://localhost:8080/ws
```
